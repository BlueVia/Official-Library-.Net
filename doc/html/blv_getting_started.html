Getting Started

Getting Started:

Environment needed:

Supported Operating Systems:

  • Windows XP (32-bit or 64-bit).
  • Windows Vista (32-bit or 64-bit).
  • Windows 7 (32-bit or 64-bit).
  • .NET Framework 4 Client Profile
  • Microsoft Visual Studio 2010
  • Also, to check your application requests is highly recommended the use of a traffic monitoring tool (as Fiddler2)

Installing the Bluevia library

First of all, you must have properly installed and configured "Microsoft Framework .NET 4" and "Microsoft Visual Studio (2010 at least)".

To check your application petitions is highly recommended the use of a traffic monitoring tool, as Fiddler2.

Once the environment is installed, download the latest Bluevia's .NET SDK from:
https://github.com/BlueVia/Official-Library-.Net

You have to ways of using the Bluevia .NET SDK:

  • By loading the Bluevia.dll in your project.

    1. Open your VisualStudio 2010, and if you are requested, select the C# configuration.

    2. Create a new console application: "File/New/Project/"

      • In the Installed Templates be sure "Visual C#" is selected.
      • ".Net Framework 4" at the selector field.
      • And select the "Console Application" option.
      • Leave the "Name", "Location" and "Solution Name" as default (ConsoleApplication1).
      • The "Create directory for solution" checkbox checked, and "Add to source control" unchecked.

    3. Probably, the solution will be configured with the ".NET Framework 4 Client Profile" by default.
      This was a problem in the old versions of the sdk (they needed the full .NET Framework 4) , but from 1.6, the Client Profile is enough;)

    4. Right now we are ready to add the .NET Bluevia's libraries to our project.

      • Right click over "References" in the "Solution Explorer" and select "Add Reference...".

      • At the "Browse" label, look for the " /lib" directory, and select "Bluevia.dll" and "Microsoft.Contracts.dll".

      • Click "OK".

      • Click on the "Refresh Button" at the "Solution Explorer", and save the changes.

    5. You are Done!!!, now you can use the Bluevia's functions.

  • Or, By adding your project in the Bluevia solution:

    1. Open the Bluevia solution: double click on "\sdk\Bluevia.sln".

    2. Once the solution is opened: you must add your project.
      Right click on the Bluevia Solution at the Solution navigator.


      Then click Add->

      • Existing Project...
        And search for your project.
      • Or a New Project...
        Create it as in the second step of the first mode..

    3. Once the Project is added to the solution, you can add the functional libraries as in the fourth step of the previous mode.

To test your program:

Save the program, and build it:

Then, you can try the program, start Debugging(F5):

You can debug step by step, by adding breaking points doubleclicking at the left side of the codeline. And go on them by "Step Into" button. (F11):

Fiddler


You can inspect your application petitions and server responses with fiddler.

Be sure to check the Decrypt HTTPS traffic option at: "Tools/Fiddler Options..."
