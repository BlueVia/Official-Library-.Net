Introduction

Introduction

Bluevia’s .NET SDK” allows you to use the BlueVia’s public API from your .NET application using just a few lines of code. You only need to download the SDK, reference the BlueVia libraries into your solution, create API keys for it, and use all the BlueVia's functionality in an easy way.

BlueVia offers you:

  • SMS sending, receiving and monitoring.
  • MMS sending, receiving and monitoring.
  • Advertise requesting.
  • User's related info receiving.
  • Location info receiving.
  • Payment capabilities.


More info:

https://bluevia.com/en/knowledge/APIs

SDK Overview:

Supported Operating Systems:

  • Windows XP (32-bit or 64-bit).
  • Windows Vista (32-bit or 64-bit).
  • Windows 7 (32-bit or 64-bit).

Environment needed:

  • .NET Framework 4
  • Microsoft Visual Studio 2010
  • Also, to check your application requests is highly recommended the use of a traffic monitoring tool (as Fiddler2)

Changes from last version:

Please pay attention to this changes:

  • GENERAL: The SDK now generates the Bluevia.dll in the \lib directory (instead than previous versions, where the .dll was generated in \src\Telefonica\bin\Release), so if you change the source code, be sure to make a copy of the original .dll file

  • GENERAL: New project full of examples for Bluevia's APIs is now added. You can load and run it from \DEMO\BlueviaExamples.

  • GENERAL: Bluevia has changed its Environment Systems to: SANDBOX, TEST and LIVE. So there are little changes to configure it.

  • GENERAL: Authenticating the client is easier, you won't need to pass the accessToken.

  • OAUTH: "RequestToken.Get" methods, now return a new "Bluevia.OAuth.Schemas.RequestToken", with an "AuthoriseUrl" field.

  • MMS MO: (Notifications and Getting messages), is now requested by 2leggedOauth Method.

  • MMS MO: "getMessage" new functionality.

  • MMS MO: new method: "GetAttachment".

  • SMS MO: (Notifications and Getting messages), is now requested by 2leggedOauth Method.

  • PAYMENT: Request parameters are now simplified.
