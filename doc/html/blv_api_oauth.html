OAUTH API GUIDE

OAUTH:

The Bluevia OAuth API is a set of functions which allows applications to retrieve, request and access tokens to complete the OAuth authentication protocol, necessary to be able to get and send data to Bluevia APIs. This guide represents a practical introduction to include the OAuth protocol in Bluevia applications.

If you want to understand how OAuth API works please read the BlueVia Bluevia's OAuth API guide.

Don't forget to read the authorization guide first.

This guide is divided into the following sections:

  • Oauth Client.
  • Oauth Methods.
    • Requesting Request Tokens.
    • Requesting Access Tokens.

 

OAuth Client:

An OAuth client represents the client side in a classic client-server schema. This object wraps up the underlying REST client side functionality needed to perform requests against a REST server.

Creating an OAuth client:
Bluevia.OAuth.Client.BV_OAuth.BV_OAuth

  • OAuth Client: 
            BV_OAuth client = new BV_OAuth(BVMode.MODE, ConsumerKey, ConsumerSecret);

 

Methods:

Requesting Request Tokens:

  • The callback parameter indicates bluevia how the verification code must be offered to the customer (oob, url, or MSISDN).

  1. Requesting Request Tokens, for OutOfBand or WebOauth verification:
    Bluevia.OAuth.Client.BV_OAuthClient.GetRequestToken 

            try
        {
                RequestToken requestToken = client.GetRequestToken
                (
                        callback: "oob" //Optional
                );
        }
        catch (BlueviaException e){}

  2. Requesting Request Tokens, for SMS verification:
    Bluevia.OAuth.Client.BV_OAuthClient.GetRequestTokenSMSHandshake 
            try
        {
                RequestToken requestToken = client.GetRequestTokenSMSHandshake
                (
                        callback: "phoneNumber" //MANDATORY
                );
        }
        catch (BlueviaException e){}

Processing the response:
Bluevia.Core.Schemas.RequestToken
The requestToken contains: requestToken key and secret, to use them to retrieve the access token, and a url to complete the user authorization proccess, and get the verification Code.

Requesting Access Tokens:
Bluevia.OAuth.Client.BV_OAuthClient.GetAccessToken

  • The oauthVerifier parameter indicates bluevia how the verification code must be offered to the customer.
  • The requestToken parameter refer to the requestToken Key previously retrieved by the getRequestToken method.
  • The requestSecret parameter refer to the requestToken Secret previously retrieved by the getRequestToken method. 
        try
        {
                Token accessToken = client.GetAccessToken
                (
                        oauthVerifier:verification, //MANDATORY
                        requestToken:null, //Optional 
                        requestSecret:null //Optional
                );
        }
        catch (BlueviaException e){}

Processing the response:
Bluevia.Core.Schemas.Token
The accessToken contains: accessToken key and secret, to use them to authenticate the clients in behalf the final user.
